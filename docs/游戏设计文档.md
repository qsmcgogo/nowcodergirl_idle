# 牛客娘挂机游戏 - 游戏设计文档

## 一、项目概述

### 1.1 游戏名称

牛客娘挂机游戏（NowCoder Girl Idle）

### 1.2 游戏类型

纯前端挂机/放置成长游戏，结合轻度战斗、主线闯关、技能树、卡片养成与后续转生循环。

### 1.3 技术栈

- **前端**：HTML5 + JavaScript + CSS
- **存储**：`localStorage`
- **架构**：按模块拆分的原生前端项目

### 1.4 当前定位

本项目当前不是传统“刷数值挂机”而已，而是希望做成：

`挂机养成 + 算法启蒙主题 + 主线闯关 + 长线构筑`

玩家一边陪牛客娘恢复“算法记忆”，一边通过挂机、练习、关卡、技能树、卡片、转生逐步变强。

### 1.5 核心玩法循环

```
挂机收集资源
-> 消耗资源解锁研究与成长功能
-> 进入练习/战斗推进当前关卡
-> 获得灵感、星星、卡片等成长收益
-> 解锁技能树、更多卡片槽与更高阶段玩法
-> 进入残响/转生循环，获得跨轮次成长
```

## 二、当前已实现内容

### 2.1 资源与挂机

当前版本已经实现以下基础资源：

| 资源名称 | 当前状态 | 说明 |
|---------|---------|------|
| **思维力** | 已实装 | 基础挂机资源，可主动切换收集 |
| **专注力** | 已实装 | 达到条件后解锁，可主动切换收集 |
| **灵感** | 已实装 | 通过战斗掉落，用于技能树等成长内容 |
| **残响** | 未实装 | 后续转生系统核心资源 |

资源特征：

- 牛客娘同一时间只能进行一种活动
- 当前可选活动包括：收集思维力、收集专注力、跑步恢复
- 思维力达到阈值后，专注力会永久显示
- 灵感第一次掉落后，将作为中期系统解锁的关键节点

### 2.2 研究与功能解锁

当前版本已实装的研究型内容：

- **练习**：消耗思维力和专注力解锁
- **跑步**：首次战败后可研究，解锁恢复期加速
- **自动练习**：达成指定条件后可研究，支持自动续战

### 2.3 战斗与练习

当前练习区已经具备完整基础循环：

- 属性栏展示
- 敌我攻击/血量展示
- 题目网格推进
- 自动挑战
- 战斗日志
- 战败恢复
- 灵感掉落

战斗是轻量回合制：

- 牛客娘与题目按回合互相造成伤害
- 当前攻击由 `思维 × 知识点 × 手速` 决定
- 当前生命由 `码力 × 耐力` 决定

### 2.4 技能树

当前版本已实装技能树第一页的两个基础分支：

- **语法**
- **数学**

它们目前会影响基础属性成长，并消耗灵感与其他资源升级。

### 2.5 存档与基础交互

当前版本已实装：

- 自动保存
- Base64 导出存档
- Base64 导入存档
- 重置数据
- 版本更新弹窗

## 三、资源系统

### 3.1 资源类型总览

| 资源名称 | 阶段 | 用途 | 主要来源 |
|---------|------|------|---------|
| **思维力** | 前期 | 解锁研究、升级、成长消耗 | 挂机收集 |
| **专注力** | 前中期 | 研究、训练、战斗相关消耗 | 挂机收集 |
| **灵感** | 中期 | 技能树、关键成长系统消耗 | 关卡/练习掉落 |
| **残响** | 后期 | 转生与跨轮次成长核心货币 | 转生结算、后续系统 |

### 3.2 当前资源产出机制

- 思维力和专注力由挂机活动稳定产出
- 灵感主要由战斗胜利掉落
- 跑步会消耗专注力并加速生命恢复

### 3.3 后续资源扩展方向

- 残响系统解锁后，星星可提供残响获得量加成
- 卡片、副本、世界 Boss 等系统可追加新的资源入口

## 四、成长系统

### 4.1 属性系统

当前属性包括：

- **思维**
- **知识点**
- **手速**
- **码力**
- **耐力**

当前公式：

- **攻击** = `思维 × 知识点 × 手速`
- **生命** = `码力 × 耐力`

后续可根据需要补充“防御”或“减伤”类属性，但要和当前生命体系统一。

### 4.2 技能树系统

技能树承担长期属性成长与系统解锁作用。

当前原则：

- 首次灵感掉落时，解锁 `关卡` 页签与 `技能树` 页签
- 技能树页签虽可见，但第一层初始为灰置
- **完成第一章全部关卡后**，正式解锁技能树第一层

设计目标：

- 让玩家提前看到长期目标
- 避免技能树在首次接触时直接“全部可点”
- 让主线闯关和数值成长建立明确因果关系

### 4.3 星星系统

星星用于章节评价与长线成长，不只是展示收集。

当前规则：

- **第一章每个关卡仅有 1 颗星**
- 通关即可获得该关卡星星
- 这样可以避免前期星数膨胀过快

星星会提供基础战斗加成，但与资源收益相关的强化不会一开始就开放，而是留到残响系统与转生技能树解锁后再逐步接入。

每颗星初始默认提供：

- 攻击 `+1%`
- 血量 `+1%`

后续在残响系统解锁后，可通过点出对应的残响技能，逐步追加：

- 灵感获得量加成
- 残响获得量加成

### 4.4 卡片系统

卡片系统是中后期的重要构筑层，和技能树互补：

- 技能树偏永久成长与结构化升级
- 卡片偏组合、搭配、养成与副本追求

#### 卡片页签解锁

- 当玩家**首次解锁任意一张卡片**时，新增 `卡片` 页签
- 在此之前，不展示卡片页签，避免前期界面过早复杂化

#### 卡片基础规则

每张卡片包含：

- 名称
- 类型
- 稀有度
- 等级
- 属性效果
- 是否已放入卡片槽

#### 卡片等级与合成

- 同名、同等级卡片，**3 张可合成 1 张高 1 级卡片**
- 卡片只保留**最高级效果生效**这一原则
- 同名卡片的低等级版本可作为合成材料继续使用

示例：

- `运算符卡片 ➕（加法·常数版）`
- 1 级属性：**收集专注力的效果提升 20%**

#### 卡片稀有度

每张卡片预设一个稀有度，用于控制：

- 掉落来源
- 获取难度
- 后续成长空间

低稀有度卡可作为基础构筑资源，高稀有度卡则来自更后期的玩法。

#### 卡片获取规则

- 主线关卡只会掉落该卡片的**第一张**
- 同名重复卡片不依赖主线反复刷出
- 额外同名卡、或更高稀有度卡片，需要通过其他系统获取
- 这些系统可以是：副本、挑战模式、活动关、转生后内容等

#### 卡片槽与生效比例

卡片放入卡片槽后，才能发挥完整效果。

当前规则：

- **已装入卡片槽**：生效 `100%` 属性
- **未装入卡片槽**：仅生效 `10%` 属性

这里的 `10%` 只是当前基准值，后续可以被技能树、转生技能或其他系统提高。

#### 卡片槽数量

- 初始只有 **1 个卡片槽**
- 后续可通过技能树、转生技能或其他高级系统增加更多卡片槽

卡片槽的存在，是卡片系统的重要策略点，避免“拿到就全部满额生效”。

## 五、主线关卡与大地图

### 5.1 总体原则

- **主线大地图保留不变**
- 关卡系统不是替代大地图，而是作为大地图内部的成长分支
- 挂机、练习、关卡、技能树、卡片、转生共同组成完整循环

### 5.2 当前已确定的解锁节奏

1. 初始以挂机和基础成长为主
2. 玩家通过战斗首次获得灵感
3. 解锁 `关卡` 与 `技能树` 页签
4. 技能树第一页保持灰置
5. 推进第一章，完成全部关卡
6. 技能树第一层点亮
7. 获得首张卡片后，解锁 `卡片` 页签
8. 中后期进入残响/转生系统

### 5.3 第一章规则

- 第一章每个关卡只有 1 颗星
- 不要求满星才能推进主线
- **第一章全部通关即可解锁技能树第一层**

### 5.4 关卡奖励定位

关卡主要承担以下职责：

- 概念教学
- 主线推进
- 灵感掉落
- 星星累计
- 首次卡片解锁

## 六、练习、战斗与副本定位

### 6.1 练习系统

练习系统是当前主循环的战斗载体：

- 推进当前题目/敌人
- 获得灵感
- 验证玩家当前成长强度
- 为后续关卡系统提供数值承载

### 6.2 副本系统定位

副本目前尚未实装，但已经有明确职能：

- 提供重复卡片来源
- 提供更高稀有度卡片来源
- 提供卡片养成材料
- 作为主线之外的长期 farm 内容

因此后续设计里：

- **主线关卡** 更偏首通、剧情、首次解锁
- **副本** 更偏重复收益、构筑追求、装备化成长

---

## 七、转生与残响系统

### 7.1 残响定义

`残响` 是后续转生系统的核心资源名称，用于承接跨轮次成长。

它代表：

- 本轮推进留下的长期痕迹
- 牛客娘恢复算法记忆后的回声
- 玩家重开后仍然保留的成长价值

### 7.2 转生系统定位

转生不是简单清档，而是新一轮成长的开始。

转生目标：

- 将本轮积累转化为残响
- 用残响购买永久增益
- 解锁新系统、新槽位、新玩法
- 让旧循环变快，新循环变深

### 7.3 残响与其他系统的关系

- 星星在残响系统解锁后，开始提供残响获得量加成
- 卡片槽数量可以由转生技能继续扩展
- 技能树可与转生技能形成“双成长线”

## 八、界面与交互规划

### 8.1 当前/规划中的主要页签

| 页签 | 状态 | 功能 |
|-----|------|------|
| **日常** | 已实装 | 资源、挂机、研究、练习入口 |
| **技能树** | 已实装/待调整规则 | 长期属性成长、层级解锁 |
| **关卡** | 规划中 | 主线章节推进、星星、首通奖励 |
| **卡片** | 规划中 | 卡片查看、合成、配置卡槽 |
| **转生/残响** | 规划中 | 跨轮次成长与永久升级 |
| **存档/版本** | 已实装 | 自动保存、导入导出、版本说明 |

### 8.2 关键交互原则

- 前期尽量简洁，不一次性展示全部系统
- 新系统优先采用“满足条件后出现页签”的方式解锁
- 玩家应始终知道“下一步目标是什么”
- 卡片、技能树、关卡的解锁顺序要能形成自然引导

## 九、技术架构

### 9.1 当前主要文件

```
nowcodergirl_idle/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── main.js         # 入口、UI 刷新、事件绑定
│   ├── game.js         # 游戏循环与初始化
│   ├── resources.js    # 资源、活动、解锁状态
│   ├── skilltree.js    # 技能树逻辑
│   ├── attributes.js   # 属性公式
│   ├── exam.js         # 当前练习/题目推进骨架
│   ├── battle.js       # 战斗、掉落、恢复、自动练习
│   ├── save.js         # 存档读写、导入导出
│   └── utils.js        # 工具函数
└── docs/
    ├── 游戏设计文档.md
    └── 算法闯关-产品设计文档.md
```

### 9.2 后续建议新增模块

```
js/
├── campaign.js     # 主线关卡/章节/星级数据
├── cards.js        # 卡片数据、合成、槽位、生效计算
├── dungeons.js     # 副本与重复收益玩法
└── rebirth.js      # 转生与残响系统
```

### 9.3 数据持久化

当前与后续需要持久化的核心数据：

- 资源数量
- 技能树等级
- 属性值
- 题目/关卡完成记录
- 灵感是否首次掉落
- 页签解锁状态
- 卡片拥有情况、卡片等级、卡片槽配置
- 星星数量
- 残响数量
- 转生次数与永久升级记录

当前存储方式：

- 浏览器 `localStorage`
- 自动保存
- Base64 导出/导入
