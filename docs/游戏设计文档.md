# 牛客娘挂机游戏 - 游戏设计文档

## 一、项目概述

### 1.1 游戏名称

牛客娘挂机游戏（NowCoder Girl Idle）

### 1.2 游戏类型

纯前端挂机/放置成长游戏，结合轻度战斗、主线闯关、技能树、卡片养成与后续转生循环。

### 1.3 技术栈

- **前端**：HTML5 + JavaScript + CSS
- **存储**：`localStorage`
- **架构**：按模块拆分的原生前端项目

### 1.4 当前定位

本项目当前不是传统“刷数值挂机”而已，而是希望做成：

`挂机养成 + 算法启蒙主题 + 主线闯关 + 长线构筑`

玩家一边陪牛客娘恢复“算法记忆”，一边通过挂机、练习、关卡、技能树、卡片、转生逐步变强。

### 1.5 核心玩法循环

```
挂机收集资源
-> 消耗资源解锁研究与成长功能
-> 进入练习/战斗推进当前关卡
-> 获得灵感、星星、卡片等成长收益
-> 解锁技能树、更多卡片槽与更高阶段玩法
-> 进入残响/转生循环，获得跨轮次成长
```

## 二、当前已实现内容

### 2.1 资源与挂机

当前版本已经实现以下基础资源：

| 资源名称 | 当前状态 | 说明 |
|---------|---------|------|
| **思维力** | 已实装 | 基础挂机资源，可主动切换收集 |
| **专注力** | 已实装 | 达到条件后解锁，可主动切换收集 |
| **灵感** | 已实装 | 通过战斗掉落，用于技能树等成长内容 |
| **残响** | 未实装 | 后续转生系统核心资源 |

资源特征：

- 牛客娘同一时间只能进行一种活动
- 当前可选活动包括：收集思维力、收集专注力、跑步恢复
- 思维力达到阈值后，专注力会永久显示
- 灵感第一次掉落后，将作为中期系统解锁的关键节点

### 2.2 研究与功能解锁

当前版本已实装的研究型内容：

- **练习**：消耗思维力和专注力解锁
- **跑步**：首次战败后可研究，解锁恢复期加速
- **自动练习**：达成指定条件后可研究，支持自动续战

### 2.3 战斗与练习

当前练习区已经具备完整基础循环：

- 属性栏展示
- 敌我攻击/血量展示
- 题目网格推进
- 自动挑战
- 战斗日志
- 战败恢复
- 灵感掉落

战斗是轻量回合制：

- 牛客娘与题目按回合互相造成伤害
- 当前攻击由 `思维 × 知识点 × 手速` 决定
- 当前生命由 `码力 × 耐力` 决定

### 2.4 技能树

当前版本已实装技能树第一页的两个基础分支：

- **语法**
- **数学**

它们目前会影响基础属性成长，并消耗灵感与其他资源升级。

### 2.5 主线关卡系统

关卡系统已从旧版 World 0 手工地图关卡，调整为更适合轻量挂机游戏的“学科分类训练”。

当前前端已实现：

- 到达主线第 2 图后解锁 `关卡` 页签
- 同时解锁第一个分类：`数学`
- 数学右侧展示 `语文` 锁定卡
- 语文提示：主线第 10 图解锁
- 关卡页展示当前等级、时间之砂、答题券、主线进度、练习/考试入口
- 时间之砂右侧加号可打开兑换弹窗
- 时间之砂可兑换全局快进或答题券
- 已新增成就页签，成就分为推图、关卡、技能树、其他、卡片、彩蛋 6 类
- 成就点会直接提升牛客娘攻击和血量，`x` 成就点提供 `+x%` 攻击和 `+x%` 血量

新规则中，每个学科拥有独立等级，初始为 F，后续通过考试晋级。数学 1-10 关已接入随机题目生成、练习、考试与答题券消耗。

旧版 Stage 0-1 天平拖拽与 World 0 节点代码暂时保留，但不再作为当前关卡系统的主方向。

### 2.6 卡片系统

当前版本已实装卡片系统基础框架：

- 卡片页签（首次获得卡片后解锁）
- 卡片持有与管理（`owned` 列表，按等级分层记录张数）
- 卡片槽系统（初始 1 个卡片槽，装入/取出）
- 卡片合成（3 张同名同级 → 1 张高 1 级）
- 卡片加成计算（槽内 100%、槽外 10%）
- 卡片详情面板（图标、名称、类型、稀有度、描述、等级效果）
- 装入槽 / 取出槽 / 一键合成按钮

当前已定义的卡片：

| 卡片 ID | 名称 | 类型 | 稀有度 | 等级上限 | 获取方式 |
|---------|------|------|-------|---------|---------|
| `equals_init` | 等号·初悟 | 运算符 | 普通（common） | 无统一上限 | Stage 0-1 首次通关 |

卡片加成采用公式计算，无等级上限（3 合 1 自然控制增长速度）：

- 公式：`baseBonus × growthRate^(等级-1)`
- 等号·初悟：基础攻击 +5%，growthRate 1.5
- 示例：1 级 +5%，2 级 +7.5%，3 级 +11.25%，4 级 +16.9%……
- 当前已扩展 `baseBonus` 字段，可支持百分比攻防、固定攻防、基础面板加值、挂机速度、回血速度、练习时间之砂等加成
- 计算顺序默认是先百分比、后固定加法；`基础攻击 +x` / `基础血量 +x` 这类字段会先加到基础面板，再参与百分比

当前稀有度标签定义：`common`（普通）、`rare`（稀有）、`epic`（史诗）、`legendary`（传说）

> 注：卡片系统设计文档中描述的颜色稀有度体系（灰/白/绿/蓝/紫/橙/红）尚未在代码中采用，当前代码使用 4 级稀有度。后续需对齐。

### 2.7 存档与基础交互

当前版本已实装：

- 自动保存（60 秒间隔）
- Base64 导出存档（支持复制和保存为 txt 文件）
- Base64 导入存档（支持粘贴和选择 txt 文件）
- 重置数据
- 版本更新弹窗
- 存档版本号管理（当前 version 7，含 campaign + cards 数据）

## 三、资源系统

### 3.1 资源类型总览

| 资源名称 | 阶段 | 用途 | 主要来源 |
|---------|------|------|---------|
| **思维力** | 前期 | 解锁研究、升级、成长消耗 | 挂机收集 |
| **专注力** | 前中期 | 研究、训练、战斗相关消耗 | 挂机收集 |
| **灵感** | 中期 | 技能树、关键成长系统消耗 | 关卡/练习掉落 |
| **答题券** | 中期 | 参加关卡考试、推动学科晋级 | 清空地图、最终格 Boss、时间之砂兑换 |
| **时间之砂** | 中期 | 快速挂机、兑换答题券 | 关卡练习结算 |
| **草稿纸** | 前中期 | 升级早期建筑 | 主线首通、刷题模式 |
| **错题笔记** | 前中期 | 升级失败补偿类建筑 | 战败、刷题模式 |
| **复习券** | 前中期 | 解锁/进入刷题模式 | 主线关键题、战败概率补偿 |
| **记忆碎片** | 中期 | 升级跨图建筑 | 每图 E/J/O/T 关键节点 |
| **残响** | 后期 | 转生与跨轮次成长核心货币 | 转生结算、后续系统 |

### 3.2 当前资源产出机制

- 思维力和专注力由挂机活动稳定产出
- 灵感主要由战斗胜利掉落
- 跑步会消耗专注力并加速生命恢复
- 草稿纸、错题笔记、复习券、记忆碎片用于建筑和刷题模式，灵感仍只服务技能树
- 答题券由清空主线地图获得，打完最终格 Boss 时额外掉落一份章节答题券
- 时间之砂由关卡练习根据练习效果产出，数学公式已确定

### 3.3 后续资源扩展方向

- 残响系统解锁后，星星可提供残响获得量加成
- 卡片、副本、世界 Boss 等系统可追加新的资源入口
- 时间之砂可用于快速挂机，或兑换答题券

## 四、成长系统

### 4.0 建筑系统

建筑用于在主线图之间提供持续解锁目标。第 1 图只解锁基础自动收益和失败补偿的一部分；第 2-5 图继续按图解锁 1-2 个建筑，后期可以改为隔几图解锁一个关键建筑。

当前节奏：

| 解锁点 | 建筑 |
|--------|------|
| 思维力/专注力启动 | 草稿桌、安静角落 |
| 首次战败 | 错题本 |
| 第 2 图 | 小憩垫、灵感灯 |
| 第 3 图 | 自动草稿机 |
| 第 4 图 | 记忆书架 |
| 第 5 图 | 备考时钟 |

建筑效果保持线性成长，成本保持指数成长，避免无上限建筑在长线中数值爆炸。

### 4.1 属性系统

当前属性包括：

- **思维**
- **知识点**
- **手速**
- **码力**
- **耐力**

当前公式：

- **攻击** = `思维 × 知识点 × 手速 × (1 + 卡片攻击加成 + 成就点加成)`
- **生命** = `码力 × 耐力 × (1 + 卡片血量加成 + 成就点加成)`

其中成就点加成为：

- `x` 成就点 = 攻击 `+x%`
- `x` 成就点 = 血量 `+x%`

后续可根据需要补充“防御”或“减伤”类属性，但要和当前生命体系统一。

### 4.2 技能树系统

技能树承担长期属性成长与系统解锁作用。

当前原则：

- 首次灵感掉落时，解锁 `技能树` 页签与技能树第一层
- 技能树不再依赖旧版“第一章全部通关”条件
- 关卡系统已经改为学科训练，不再把章节清空作为技能树解锁前置
- 每一层解锁前只显示该层锁定块，鼠标悬浮展示解锁条件；解锁后显示该层内容，并显示下一层锁定块

设计目标：

- 让灵感第一次掉落成为中期成长系统的明确开启点
- 避免新关卡体系继续绑定旧章节结构
- 让玩家拿到灵感后立刻知道它可以用于长期成长

当前结构：

| 层级 | 节点 | 解锁/前置 | 每级效果 |
|------|------|-----------|----------|
| 第一层 | 语法 | 首次获得灵感 | 知识点 +0.1，手速 +0.1，耐力 +1 |
| 第一层 | 数学 | 首次获得灵感 | 思维 +0.2，知识点 +0.2 |
| 第二层 | 顺序结构 | 语法、数学均达到 Lv.5 | 手速 +0.08，耐力 +0.5 |
| 第二层 | 变量类型 | 语法 Lv.10 | 知识点 +0.15，码力 +0.05 |
| 第二层 | 加法 | 语法、数学均达到 Lv.5 | 思维 +0.15，知识点 +0.05 |
| 第二层 | 减法 | 加法 Lv.5 | 手速 +0.12，耐力 +0.8 |

视觉结构：

- `语法` 向左下连接 `顺序结构`
- `语法` 向右下连接 `变量类型`
- `数学` 向下连接 `加法`
- `加法` 向右连接 `减法`

### 4.3 星星系统

星星是旧版章节关卡的评价资源。关卡系统改为学科训练后，星星暂不作为新关卡的主要奖励口径。

当前处理原则：

- 旧版星星逻辑可暂时保留，不作为新关卡扩展方向
- 新关卡优先使用时间之砂、答题券、普通卡、稀有卡作为奖励
- 若后续恢复星星，应重新定义其来源与作用，而不是继续绑定旧版第一章

后续如果星星回归，可考虑作为残响、转生或高阶关卡评价资源。

### 4.4 卡片系统

卡片系统是中后期的重要构筑层，和技能树互补：

- 技能树偏永久成长与结构化升级
- 卡片偏组合、搭配、养成与副本追求

#### 卡片页签解锁

- 当玩家**首次解锁任意一张卡片**时，新增 `卡片` 页签
- 在此之前，不展示卡片页签，避免前期界面过早复杂化

#### 卡片基础规则

每张卡片包含：

- 名称
- 类型
- 稀有度
- 等级
- 属性效果
- 是否已放入卡片槽

#### 卡片等级与合成

- 同名、同等级卡片，**3 张可合成 1 张高 1 级卡片**
- 卡片只保留**最高级效果生效**这一原则
- 同名卡片的低等级版本可作为合成材料继续使用

示例：

- `运算符卡片 ➕（加法·常数版）`
- 1 级属性：**收集专注力的效果提升 20%**

#### 卡片稀有度

每张卡片预设一个稀有度，用于控制：

- 掉落来源
- 获取难度
- 后续成长空间

低稀有度卡可作为基础构筑资源，高稀有度卡则来自更后期的玩法。

#### 卡片获取规则

- 主线关卡只会掉落该卡片的**第一张**
- 同名重复卡片不依赖主线反复刷出
- 额外同名卡、或更高稀有度卡片，需要通过其他系统获取
- 这些系统可以是：副本、挑战模式、活动关、转生后内容等

#### 卡片槽与生效比例

卡片放入卡片槽后，才能发挥完整效果。

当前规则：

- **已装入卡片槽**：生效 `100%` 属性
- **未装入卡片槽**：仅生效 `10%` 属性

这里的 `10%` 只是当前基准值，后续可以被技能树、转生技能或其他系统提高。

#### 卡片槽数量

- 初始只有 **1 个卡片槽**
- 后续可通过技能树、转生技能或其他高级系统增加更多卡片槽

卡片槽的存在，是卡片系统的重要策略点，避免“拿到就全部满额生效”。

### 4.5 成就系统

当前已实装成就系统基础框架。

成就分类：

- 推图
- 关卡
- 技能树
- 其他
- 卡片
- 彩蛋

展示规则：

- 成就入口位于底部菜单，打开后以全屏弹窗展示。
- 初始只展示 `推图` 成就页签。
- 其他分类页签会随相关系统解锁后出现，例如关卡系统解锁后出现 `关卡`，卡片系统解锁后出现 `卡片`。
- `关卡` 分类内的语文相关成就需要主线第 10 图解锁语文后才显示。
- 除彩蛋外，已展示的未获取成就可以查看名称和细节。
- 彩蛋成就未触发前不显示真实名称和细节，只显示隐藏成就占位。
- 推图成就采用链式展示，初始只展示“第一图通关”成就；获取后再展示下一项。
- 获得成就时，右下角出现小弹窗提示，5 秒后自动消失。

推图成就当前命名：

| 条件 | 成就名 |
|------|--------|
| 第 1 图通关 | 微光 |
| 第 2 图通关 | 第二步 |
| 第 3 图通关 | 三点成线 |
| 第 5 图通关 | 小径成章 |
| 第 10 图通关 | 十图远行 |
| 第 20 图通关 | 二十连峰 |
| 第 30 图通关 | 长风三十 |
| 第 50 图通关 | 半百征途 |
| 第 100 图通关 | 百图之证 |

成就名后续可以继续调整，不影响系统结构。

## 五、主线关卡与大地图

### 5.1 总体原则

- 关卡系统采用学科分类，而不是每关单独绘制复杂交互
- 主线用于控制学科解锁节奏
- 练习不限次数，考试消耗答题券并用于晋级
- 挂机、练习、关卡、技能树、卡片、转生共同组成完整循环

### 5.2 当前已确定的解锁节奏

1. 初始不展示关卡系统
2. 到达主线第 2 图后，解锁 `关卡` 页签与 `数学` 分类
3. 数学页右侧展示 `语文` 锁定卡，提示主线第 10 图解锁
4. 到达主线第 10 图后，解锁 `语文` 分类
5. 后续继续扩展算法等新分类
6. 中后期进入卡片、副本、残响/转生等长期系统

### 5.3 练习与考试

- 练习：60 秒连续做题，不限次数
- 练习时可选择当前段位或低于当前段位的题目
- 考试：60 秒内答对 8 题
- 考试答错会扣除 5 秒剩余时间，并立刻更换题目
- 考试开局会批量生成一组随机题目，避免固定题表被背题
- 考试成功后该学科晋级
- 考试需要答题券
- 练习根据练习效果产出时间之砂
- 数学练习根据正确题数掉落普通卡片：`x <= 5` 为 0 张，`6 <= x <= 10` 为 1 张，`x > 10` 为 `floor((x / 10) ^ 5)` 张
- 数学练习时间之砂公式：`floor(关卡等级数值 ^ 2 * 正确题数 ^ 1.5)`，F=1，E=2，以此类推

数学前 10 关当前已定义到幂运算，第 11 关及以后后续再设计。数学题目由每关随机数生成器生成，并按当前关卡混入旧关卡题型：当前关卡 50%，前一关 25%，继续向前按一半递减，第 1 关承接剩余概率。减法题会保证不出现负数。

### 5.4 关卡奖励定位

关卡奖励当前准备重做，先不绑定旧版星星和首通卡片逻辑。

当前已确定的奖励资源方向：

- 清空一个主线地图后获得答题券
- 打完该地图最终格 Boss 时，额外掉落一份章节答题券
- 每次练习根据练习效果获得时间之砂
- 数学练习会按正确题数掉落普通卡片，用于后续合成
- `10000` 时间之砂可兑换 `10` 分钟全局快进
- `10000` 时间之砂可兑换 `1` 张答题券
- 快进时间按线性比例折算，即 `1000` 时间之砂兑换 `1` 分钟

全局快进当前会结算：

- 挂机资源
- 主线图对战

快进结算使用 JSON 结构，后续可继续追加更多系统收益块。

新关卡主要承担以下职责：

- 提供主动答题玩法
- 提供学科等级目标
- 承接主线解锁节奏
- 为后续奖励、卡片、副本、转生系统提供入口

## 六、练习、战斗与副本定位

### 6.1 练习系统

练习系统是当前主循环的战斗载体：

- 推进当前题目/敌人
- 获得灵感
- 验证玩家当前成长强度
- 为后续关卡系统提供数值承载

### 6.2 副本系统定位

副本目前尚未实装，但已经有明确职能：

- 提供重复卡片来源
- 提供更高稀有度卡片来源
- 提供卡片养成材料
- 作为主线之外的长期 farm 内容

因此后续设计里：

- **主线关卡** 更偏首通、剧情、首次解锁
- **副本** 更偏重复收益、构筑追求、装备化成长

---

## 七、转生与残响系统

### 7.1 残响定义

`残响` 是后续转生系统的核心资源名称，用于承接跨轮次成长。

它代表：

- 本轮推进留下的长期痕迹
- 牛客娘恢复算法记忆后的回声
- 玩家重开后仍然保留的成长价值

### 7.2 转生系统定位

转生不是简单清档，而是新一轮成长的开始。

转生目标：

- 将本轮积累转化为残响
- 用残响购买永久增益
- 解锁新系统、新槽位、新玩法
- 让旧循环变快，新循环变深

### 7.3 残响与其他系统的关系

- 星星在残响系统解锁后，开始提供残响获得量加成
- 卡片槽数量可以由转生技能继续扩展
- 技能树可与转生技能形成“双成长线”

## 八、界面与交互规划

### 8.1 当前/规划中的主要页签

| 页签 | 状态 | 功能 |
|-----|------|------|
| **日常** | 已实装 | 资源、挂机、研究、练习入口 |
| **技能树** | 已实装/待调整规则 | 长期属性成长、层级解锁 |
| **关卡** | 已实装（轻量学科页） | 数学训练入口、语文锁定预告、数学练习/考试 |
| **卡片** | 已实装（基础框架） | 卡片查看、合成、配置卡槽、装备加成 |
| **成就** | 已实装（基础框架） | 成就点加成、分类展示、推图链式展示、彩蛋隐藏 |
| **转生/残响** | 规划中 | 跨轮次成长与永久升级 |
| **存档/版本** | 已实装 | 自动保存、导入导出、版本说明 |

### 8.2 关键交互原则

- 前期尽量简洁，不一次性展示全部系统
- 新系统优先采用“满足条件后出现页签”的方式解锁
- 玩家应始终知道“下一步目标是什么”
- 卡片、技能树、关卡的解锁顺序要能形成自然引导
